# Find-Repos 链路收敛与 Stars 展示修复计划

## Summary

本次改动聚焦两件事：把 `find-repos` 主链路收敛到“按 TTL due、按 surface 顺序短路、没找到就继续”的最小心智模型；修复 paper table 中 `Repo` 不为空但 `Stars` 为空的大量误展示。

## Key Changes

- AlphaXiv API 和 AlphaXiv HTML 拆成同级独立 surface：API 找到 URL 才短路；API 没找到、404、500、timeout、429 都继续查 HTML；HTML 没找到再继续 Hugging Face。
- 保留最小可观测性：非 404 provider 错误仍写 `fetch_failed` observation 和 failure metrics；`last_attempt_complete` 可继续表示 provider 链路是否完整，但不再主导“业务是否完成”。
- item-level resume 收敛为稳定结果维度：持久化后只要 paper 形成 `found`、`ambiguous` 或 `not_found`，就记录 `resume_items_completed`；只有 `unknown` 不记录 completed。
- Due/TTL 主干保持克制：`force=true` 全部查；`force=false` 只查没有 TTL 或 TTL 到期的 paper。`unknown` 和没有 `repo_state` 只是“没有 TTL”的实现形态，不作为新的产品心智。
- `/papers` summary 增加轻量字段 `primary_repo_stars: int | null`，后端按 `primary_repo_url` 查询 `GitHubRepo.stars` 返回；前端 Stars 列直接使用该字段，不再依赖 `/repos?limit=10000` 的不完整 join。

## Implementation Changes

- `find-repos` provider 链路：调整 `_probe_alphaxiv`，去掉 API 404/错误对 HTML 的提前 return；API 错误只追加 `fetch_failed` observation，然后继续 HTML。
- `find-repos` resume：在 `run_find_repos` 持久化后，按 `state.stable_status in {found, ambiguous, not_found}` 记录 item resume completed，而不是按 `result.complete`。
- API/schema/frontend：扩展 `PaperSummaryRead` 和 paper summary payload；前端 `PaperSummary` type 增加 `primary_repo_stars`，`paperRows.repo_stars` 改用该字段。
- 不改 `/repos` preview limit，不改 dashboard 统计口径，不删除 `unknown` / `ambiguous` / `last_attempt_complete` / observations。

## Test Plan

- 更新现有 AlphaXiv 404 测试：API 404 后必须请求 HTML；HTML 也没找到时，完整链路可写 `not_found + TTL`。
- 新增 AlphaXiv API 404 + HTML found 测试：最终 `found`，设置 TTL，记录 API checked-no-match 和 HTML found。
- 新增 AlphaXiv API 非 404 错误 + HTML found 测试：最终 `found`，设置 TTL，保留 API `fetch_failed`，并记录 item resume completed。
- 新增 resume 测试：稳定 `found/ambiguous/not_found` 会 completed；`unknown` 不 completed。
- 新增 `/papers` API 测试：有 `GitHubRepo` metadata 时返回 `primary_repo_stars`；没有 metadata 或 GitHub 404 时返回 `null`。
- 跑 `uv run python -m pytest`、`npm run build`、`git diff --check`。`npm run lint` 如仍失败，只报告既有 lint 问题，不顺手重构无关前端生命周期代码。

## Assumptions

- `last_attempt_complete=false` 继续作为观测字段保留，不影响稳定结果 TTL 和 item-level resume。
- `ambiguous` 保留，含义是“第一个命中的 surface 内提取到多个不同 repo URL”。
- `primary_repo_stars` 是本次唯一新增 paper summary 字段；不新增批量 repo metadata 接口。
