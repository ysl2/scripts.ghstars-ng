# 统一 Batch / Child Job 生命周期模型

## Summary

本次把 batch job 明确定义为 child-centric 模型：batch folder 是聚合视图和批量操作入口，child job 是实际执行单元。主 job 的 `Stop`/`Re-run` 影响所有 child；child 的 `Stop`/`Re-run` 只影响自身。历史上的 parent `stop_requested_at` 不再永久禁止 child rerun，也不再永久压制 batch folder 的显示状态。

## Key Changes

- 后端重定义 batch folder display state：`stopping > running > queued > failed > cancelled > succeeded`，其中 active child 优先，所有 latest child 都终态后再按 terminal 最差状态显示。
- 后端 child rerun 只在 parent 当前显示 `stopping` 时拒绝；parent 已稳定 `cancelled`/`failed`/`succeeded` 后，允许单独 rerun finished child。
- 后端保留现有去重：同一 child scope 如果已有 active job，仍拒绝重复 rerun。
- 前端按钮逻辑与后端一致：child `Stop` 只看 child 自己是否 active；child `Re-run` 只在 parent 当前 `stopping` 时禁用，不因 parent 历史 stop/cancel 禁用。
- 当前未提交的保守修复需要收窄：移除 parent `stop_requested_at` / parent raw `cancelled` 对 child rerun 的永久阻断。
- batch folder `Re-run` 改为原地执行：不创建新的 batch root，只在当前 parent 下为 retryable scopes 创建新的 child repair attempts。

## API / Interface

- 不新增字段，不改 schema，不做 migration。
- `JobRead.batch_state` 的语义会变化：batch folder 状态由 latest child states 主导，历史 `stop_requested_at` 只作为详情字段保留，不再永久决定显示状态。
- 现有 `POST /api/v1/jobs/{id}/rerun` 继续用 `409 Conflict` 表达不可 rerun；新的 409 条件是 parent 当前 `stopping` 或已有同 scope active job。
- `POST /api/v1/jobs/{batch_id}/rerun` 对 batch root 返回同一个 `JobRead.id`，不再返回新的 batch root id。

## Test Plan

- 参数化覆盖 `sync_papers_batch`、`find_repos_batch`、`refresh_metadata_batch` 的 batch display state：active child 优先，terminal 状态按 `failed > cancelled > succeeded`。
- 覆盖 parent stop 后所有 active children 被 stop/cancel，parent 稳定显示 `cancelled`。
- 覆盖 parent 稳定 `cancelled` 后，单独 rerun 某个 cancelled child 会成功；child pending/running 时 parent 显示 `queued`/`running`；child 完成后 parent 回到 terminal 最差状态。
- 覆盖 parent 当前 `stopping` 时，finished child rerun 返回 `409`。
- 覆盖 parent 当前 `running` 但不是 `stopping` 时，另一个 finished child 可以 rerun，前提是同 scope 没有 active duplicate。
- 覆盖 batch folder 原地 rerun：parent id 不变，只为 failed/cancelled/missing scopes 创建 child repair attempts；succeeded scopes 保持原样。
- 跑 `uv run python -m pytest tests/test_job_batching.py -q`、完整 `uv run python -m pytest -q`、`npm --prefix frontend run build`。

## Assumptions

- 长期保持单 worker，不设计多 worker 并发语义。
- `export` 不是 batch/child 类型，本次不改变它的 stop/rerun 支持范围。
- batch folder `Re-run` 仍只对 finished batch 有意义；active batch 使用 `Stop`，不支持 active batch root rerun。
