# Fix expanded chevron loading flicker

## Context

Batch folder rows can remain expanded while their child jobs are being refreshed. The current chevron renderer uses mutually exclusive classes for `loading` and `expanded`, so the expanded rotation class is briefly removed during loading and the icon flickers as collapsed.

## Plan

1. Update the batch child chevron renderer so `expanded` and `loading` are additive classes.
2. Apply the same additive class handling to the attempt history chevron renderer, because it has the same state shape.
3. Run the frontend build to verify the TypeScript and Vite bundle still pass.

## Expected Behavior

- Collapsed and idle rows keep only the base toggle class.
- Collapsed and loading rows get the base class plus `loading`.
- Expanded and idle rows get the base class plus `expanded`.
- Expanded and loading rows get the base class plus both `expanded` and `loading`, preserving the visual expanded direction while loading.
