# Stabilize expanded job chevron during loading

## Context

Expanded batch folder rows refresh their children while the Jobs table is polling. During each refresh, `loading` is briefly added and removed from the chevron button. The expanded direction is now preserved, but the loading CSS still changes the button background and icon opacity, which can make the expanded icon visibly blink.

## Plan

1. Keep the existing loading state and ARIA behavior unchanged.
2. Add an expanded-loading CSS override so expanded chevrons keep the expanded visual treatment while child rows or history rows are loading.
3. Apply the same override to attempt history chevrons because they share the same expanded/loading state model.
4. Run the frontend build to verify the change.

## Expected Behavior

- A collapsed row that is loading can still use the loading visual.
- An expanded row that is loading remains visually stable as expanded.
- The chevron should no longer blink when polling refreshes child jobs or attempt history.
