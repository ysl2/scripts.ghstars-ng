# Papers table GitHub updated column implementation plan

## Summary

Add a GitHub repository `Updated` column to the frontend Papers table and move `Journal Ref` next to `Description`, while continuing to reuse the existing paginated `/api/v1/papers` summary path.

## Implementation

- Extend `PaperSummaryRead` and `/api/v1/papers` summary rows with `primary_github_updated_at` from `github_repos.updated_at`.
- Keep repository metadata lookup batched per `/papers` page; do not add frontend requests or depend on `/repos`.
- Update frontend `PaperSummary`, row mapping, and Papers column definitions to include `Updated` immediately after `Pushed`.
- Move `Journal Ref` to the left of `Description` in the Papers table.
- Bump the Papers grid persistence id to avoid stale localStorage column ordering.

## Tests

- Update public API tests for present, missing, and paged repository metadata.
- Run backend public API tests and frontend build.
