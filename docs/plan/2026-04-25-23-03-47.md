# Find-Repos External Surface Refactor Plan

## Summary

把 `find-repos` 外部 source 结构收敛成同级 surface：`AlphaXiv API`、`AlphaXiv HTML`、`Hugging Face API` 由主链路统一按顺序调度，避免 AlphaXiv API/HTML 在代码结构上继续被绑定成特殊 provider 内部流程。

## Key Changes

- 拆除 `_probe_alphaxiv()` 组合 helper，改为独立 `_probe_alphaxiv_api()` 和 `_probe_alphaxiv_html()`。
- `_lookup_links_for_paper()` 统一执行短路链路：comment、abstract、AlphaXiv API、AlphaXiv HTML、Hugging Face API。
- 保持现有行为不变：任一 surface 找到 repo 后停止后续 surface；非 404 错误记录 `fetch_failed` 和 failure metrics，但当前 surface 没找到时继续下一个 surface。
- 保持现有 metrics/raw_fetch/observations wire shape 不变，不改 API schema，不改前端。

## Test Plan

- 更新/补充测试，确保 AlphaXiv API 找到时不会查 HTML/Hugging Face。
- 保留现有测试，确保 AlphaXiv API 404 或错误时继续查 HTML，HTML 找到后跳过 Hugging Face。
- 跑 `uv run python -m pytest`、`npm run build`、`git diff --check`。
